# Documentation

This documentation follows the Diataxis framework.

## Quadrants

| Quadrant | Purpose | Location |
| --- | --- | --- |
| Tutorials | Guided learning paths to build confidence from zero to a working result | `docs/tutorials/` |
| How-to Guides | Task-focused procedures to solve a specific operational objective | `docs/how-to-guides/` |
| Reference | Authoritative facts, specifications, schemas, and parameter tables | `docs/reference/` |
| Explanation | Conceptual context, rationale, tradeoffs, and design decisions | `docs/explanation/` |

## Authoring Rules

| Rule | Requirement |
| --- | --- |
| Intent purity | Each document belongs to exactly one quadrant |
| Cross-quadrant linking | Link to related docs instead of mixing intents |
| Terminology | Use stable infrastructure terms consistently |
| Accuracy | Keep documentation aligned with implemented behavior and current topology |
